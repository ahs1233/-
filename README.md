# السوگ (Al-Souq)

منصة تجارة إلكترونية **متعددة البائعين** للسوق العراقي — تجمع التجار التقليديين
وبائعي السوشيال ميديا في واجهة عربية واحدة (RTL أولاً)، مع **الدفع عند الاستلام (COD)**.

> الحالة: **المراحل 1–6 مكتملة** — الواجهات الثلاث + **الإشعارات (Web Push)** +
> **البحث العربي المحسّن** (متعدّد الكلمات + اقتراحات فورية) + **PWA كامل**
> (service worker بطبقات تخزين، تثبيت، استقبال Push).

---

## المعمارية

Monorepo (Turborepo + pnpm) يفصل منطق الأعمال عن طبقة العرض، بحيث يُعاد استخدام
نفس الـ backend لتطبيق native (React Native / Expo) لاحقاً دون تغيير.

```
apps/
  web/                 Next.js 14 (App Router) + Tailwind RTL + PWA
packages/
  db/                  Prisma schema + migrations + seed (بيانات عراقية)
  api/                 راوترات tRPC (الطبقة المنطقية) + توليد OpenAPI/REST
  domain/              منطق أعمال نقي (آلة حالة الطلب، العمولة، المخزون)
  auth/                OTP + JWT + جلسات + RBAC + تجريد مزوّد SMS
  validators/          مخططات Zod مشتركة (عميل/خادم/native)
  i18n/                الرسائل (ar افتراضي، en/ku جاهزان للإضافة)
  ui/                  مكوّنات واجهة واعية بالـ RTL
  config/              design tokens + tailwind preset + tsconfig
  utils/               تطبيع عربي + تنسيق IQD + أرقام هاتف عراقية
```

### قرارات أساسية
- **العربية RTL أولاً**؛ i18n جاهز للإنجليزية/الكردية.
- **الدفع عند الاستلام** أساس، وبنية مفتوحة لإضافة ZainCash / FIB / Qi.
- **PWA**: قابلة للتثبيت، قشرة أوفلاين أساسية (التوسّع في المرحلة 6).
- **العملة**: الدينار العراقي (IQD) بتنسيق أرقام عربي صحيح.
- **العقد**: tRPC للويب (type-safe) + توليد **OpenAPI** تلقائياً للـ native.

## التشغيل محلياً

### المتطلبات
- Node.js ≥ 18.18، pnpm ≥ 9، Docker (لقاعدة البيانات والتخزين).

### الخطوات
```bash
# 1) المتغيّرات
cp .env.example .env        # عدّل الأسرار

# 2) البنية المحلية (PostgreSQL + MinIO)
docker compose up -d

# 3) التبعيات
pnpm install

# 4) قاعدة البيانات + البذور
pnpm db:generate
pnpm db:migrate             # يطبّق migrations الرسمية (prisma/migrations)
pnpm db:seed                # بيانات عراقية + مراجعات + طلبات نموذجية

# 5) التشغيل
pnpm dev                    # http://localhost:3000
```

### حسابات البذور (مزوّد SMS = console)
عند تسجيل الدخول يُطبع رمز OTP في سجل الخادم (ويظهر في صفحة الدخول أثناء التطوير).

| الدور | الهاتف |
|------|--------|
| أدمن | `07700000000` |
| بائع | `07701111111` … `07704444444` |

## العقود للـ native
- وثيقة OpenAPI: `GET /api/openapi.json` (15 endpoint موثّق للمشتري)
- جسر REST: `/api/rest/*` (نفس منطق tRPC)
- نفس مصادقة OTP ونفس نماذج البيانات.
- تفاصيل المعمارية والعقود: [`ARCHITECTURE.md`](./ARCHITECTURE.md)

## الإشعارات (Web Push)
```bash
pnpm vapid    # ولّد مفاتيح VAPID وضعها في .env (عام + خاص + NEXT_PUBLIC_*)
```
تبقى الإشعارات داخل التطبيق تعمل دائماً؛ Push يُفعّل عند ضبط المفاتيح.

## النشر

📘 **الدليل الكامل خطوة بخطوة**: [`DEPLOYMENT.md`](./DEPLOYMENT.md)

### الأسهل: نقرة على Render (قاعدة بيانات + موقع معاً)
[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy)

Render يقرأ `render.yaml` فيُنشئ القاعدة + الموقع + يطبّق الهجرات والبيانات تلقائياً.
رمز الدخول يظهر على الشاشة (وضع `console`) فتسجّل الدخول فوراً من هاتفك.

### بديل: Vercel + Neon
- اربط المستودع (Root = الجذر)؛ الإعدادات من `vercel.json`.
- اضبط `DATABASE_URL`, `DIRECT_URL`, `JWT_*` (واختيارياً `S3_*`, `VAPID_*`).
- طبّق الهجرات مرة واحدة: `pnpm db:deploy && pnpm db:seed` ضد قاعدة Neon.

### الويب (Docker — أي مزوّد)
```bash
docker build -t al-souq-web .
docker run -p 3000:3000 --env-file .env al-souq-web
```
يستخدم إخراج Next standalone لصورة صغيرة.

### قاعدة البيانات والتخزين
- PostgreSQL مُدار (Render/Neon/Supabase) — طبّق الهجرات: `pnpm db:migrate deploy`.
- تخزين S3-compatible (Cloudflare R2/MinIO/S3) لصور المنتجات.

### CI
`.github/workflows/ci.yml` يشغّل على كل دفعة: install → generate → push + seed →
typecheck → lint → اختبارات (وحدة + تكامل على PostgreSQL) → build.
