# معمارية السوگ (Al-Souq)

وثيقة مرجعية للبنية والعقود والقرارات. الهدف: فصل منطق الأعمال عن طبقة العرض
بحيث يُعاد استخدام نفس الـ backend لتطبيق native (React Native/Expo) لاحقاً.

## 1. الطبقات

```
┌─────────────────────────────────────────────────────────────┐
│  العرض (Delivery)                                            │
│  apps/web — Next.js App Router: (shop) / (vendor) / (admin)  │
│  + (مستقبلاً) apps/mobile — Expo يستهلك نفس العقد            │
└───────────────┬─────────────────────────────────────────────┘
                │ tRPC (نوعي) + REST/OpenAPI (native)
┌───────────────▼─────────────────────────────────────────────┐
│  العقد (API) — packages/api                                  │
│  راوترات tRPC + سياق + RBAC + توليد OpenAPI                  │
│  services/: order, vendor-less helpers, admin, push          │
└───────────────┬─────────────────────────────────────────────┘
                │
┌───────────────▼──────────────┐  ┌──────────────────────────┐
│  منطق أعمال نقي              │  │  أدوات وعقود مشتركة       │
│  packages/domain             │  │  validators (Zod)        │
│  - آلة حالة الطلب            │  │  utils (تطبيع عربي/IQD)   │
│  - حساب العمولة              │  │  i18n · ui · config      │
│  - منطق المخزون والحجز       │  │  auth · storage          │
└───────────────┬──────────────┘  └──────────────────────────┘
                │
┌───────────────▼─────────────────────────────────────────────┐
│  البيانات — packages/db (Prisma + PostgreSQL)               │
└─────────────────────────────────────────────────────────────┘
```

**قاعدة الاعتماد:** الأعلى يعتمد على الأدنى فقط. `domain` لا يعرف Prisma ولا Next،
لذا قواعده قابلة للاختبار بمعزل ومشتركة عبر المنصات.

## 2. العقد للـ native

- **tRPC** للويب: أمان نوعي كامل من الخادم للعميل دون توليد كود.
- **REST/OpenAPI** للـ native: نفس الراوترات تُصدَّر تلقائياً عبر `trpc-to-openapi`.
  - الوثيقة: `GET /api/openapi.json`
  - الجسر: `/api/rest/*`
  - المغطّى حالياً: المصادقة، الجغرافيا، الكتالوج والبحث، الطلبات، العناوين، المفضلة.
- **المصادقة موحّدة**: OTP عبر الهاتف العراقي → JWT access + refresh session.
  - الويب: كوكيز httpOnly. الـ native: نفس التوكنات في جسم الرد + ترويسة `Authorization`.
- **التصميم**: design tokens في `packages/config` (ألوان/مسافات/خطوط) قابلة لإعادة
  الاستخدام في Expo عبر نفس القيم.

## 3. منطق الأعمال الحرج

### آلة حالة الطلب (`domain/order-state`)
```
PENDING → CONFIRMED → PREPARING → SHIPPED → DELIVERED → COMPLETED
        ↘ CANCELLED (قبل الشحن)         ↘ RETURNED (بعد الشحن/التسليم)
```
كل انتقال مقيّد بالدور (CUSTOMER/VENDOR/ADMIN/SYSTEM). الأدمن يتجاوز قيود الدور
لا قيود صحّة الانتقال.

### المخزون والـ COD (`domain/stock` + `api/services/order`)
- لا يُحجز مبلغ في COD، لكن يُحجز المخزون فور الطلب: `reservedStock += qty`.
- **منع التزامن**: تحديث SQL شرطي ذرّي
  `UPDATE ... SET reservedStock = reservedStock + q WHERE stock - reservedStock >= q`
  فإن لم يتأثر صف → نفاد المخزون (آمن تحت الطلبات المتزامنة).
- **دورة الحياة**: محجوز حتى التسليم → عند `DELIVERED` يُخصم من `stock` فعلياً
  ويزيد `soldCount`؛ عند الإلغاء يُحرَّر الحجز؛ عند الإرجاع بعد التسليم يُعاد للمخزون.
- حجوزات منتهية الصلاحية لطلبات PENDING تُحرَّر وتُلغى تلقائياً.

### العمولة والتسوية
- العمولة تُحسب على مجموع المنتجات (لا التوصيل) وتُثبَّت لحظة الطلب.
- التسوية: الأدمن يجمع العمولات غير المسوّاة لطلبات مسلّمة/مكتملة → `Payout` مدفوع
  مربوط بتلك العمولات (ذرّياً)، مع إشعار وسجل تدقيق.

## 4. الأمان

- **RBAC** على كل endpoint عبر إجراءات tRPC (`protected/vendor/admin`).
- **حماية IDOR**: كل وصول لمورد يتحقّق من الملكية (طلب/عنوان/منتج بائع).
- **التحقّق**: Zod على كل المدخلات (مشترك عميل/خادم).
- **الجلسات**: refresh مجزّأ (sha256)، access قصير (JWT)، إبطال عند الحظر.
- **سجل التدقيق**: كل إجراء إداري حسّاس يُسجَّل (من، ماذا، قبل/بعد).
- **OTP**: رمز مجزّأ (bcrypt)، حدّ محاولات، انتهاء صلاحية، استخدام واحد.

## 5. البحث العربي

تطبيع موحّد (`utils/arabic`): إزالة التشكيل، توحيد الألف/الياء/التاء/الهمزات،
تحويل الأرقام. يُخزَّن `titleNorm` ويُطابَق بحثاً متعدّد الكلمات (AND) بصرف النظر
عن الترتيب، مع اقتراحات فورية.

## 6. الاختبارات

- **وحدة** (`domain`, `utils`, `auth`): آلة الحالة، العمولة، المخزون، التطبيع،
  الهاتف، العملة، RBAC.
- **تكامل** (`api/test`): دورة حياة الطلب على قاعدة بيانات حقيقية، تدفّقات المصادقة
  والمشتري والبائع والأدمن، وحماية RBAC/IDOR.
- CI يشغّلها على PostgreSQL حقيقي (GitHub Actions).
